Each recipe file contains

1

When to use

Signals in the user's request that mean "use this type" — e.g., "show the flow", "document the API calls", "map the states"

2

Layout template

Canonical x/y coordinates, spacing rules, zone layout — Claude fills in node labels and connections without computing coordinates from scratch

3

Color & style defaults

Which semantic colors to use per node role, arrow types, font sizes — consistent with color-palette.md and layout-rules.md

4

Pitfalls + worked example

Common mistakes for this type (e.g., bidirectional arrows in sequence, overlap in mindmap) and one full bash script example
